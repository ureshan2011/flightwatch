☁ How this maps to Firebase (what's real vs simulated) ▾
This prototype runs in demo mode — trips live in localStorage and alerts are simulated. Set window.FARO_FIREBASE to a real config and the same code paths become:
• Identity → Firebase Auth (anonymous uid, upgradable to Google).
• Pinned trips → Firestore users/{uid}/trips, synced across devices via onSnapshot.
• Alert toggles + price target → fields on each trip doc, read by a scheduled Cloud Function that pushes FCM / Telegram / email when your trip hits BUY, a new low, or a closing window.
• "Simulate a new scan" stands in for the scan writing fresh verdicts to routes/* — which would live-update these cards. See redesign/FIREBASE.md.
